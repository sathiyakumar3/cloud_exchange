Sites - Your Digital Folders

Fundamentally you will need a site - technically, a folder to place all your devices. However, a Public Folder is readily available in navigation tree, called Cloud_Exchange However please be advised this is a community folder, which means your data is not secure. It is only recommended for testing or learning. Though It is not necessary to create a new site for each user, you can request access for existing sites, for which the approvals are required from the site administrators or the owners. Administrators play a crucial part in maintaining the sites. Please also note that the public site : Cloud_Exchange has no owners, or administrators.

Every user must select the role he or she is to play in every Site.

  • Member– Monitoring of all devices
  • Developer– Monitoring and Management of all devices.
  • Administrator– Monitoring and Management of all users and devices.
  • Owner– Owns of the site, Monitoring and Management of all users, devices and the site itself.

Generally, the Member is the user to whom the collected data is intended for, hence all the implementation part is played by the administrators and developers. Administrator generally controls the users’ access to the site. The site access requests are directed to the administrator for approvals. Even after the approval, the administrator has rights to revoke access to the site at any given time.

Developer on the hand, has access to the control of the devices in a site. A developer is responsible for writing the middleware to get the intended data to the cloud. How to write a middleware to get the intended data to the cloud?

Devices - Connecting to Cloud

This is technically the device that is being connected to the cloud exchange. The device is identified by the following parameters.

  • Device ID– It is the unique ID that you are going to identify your device.
  • Registry ID – It is basically the site name, under which you are expecting the device to be placed.

Along with the above credentials, the following parameters are also required. This is mainly for security.

  • Public Key – This is usually generated in key pairs. Public Key and Private Key. Public keys are generally opened to the public, as the name suggests. On the other hand, the Private Key must remain confidential to its respective owner. Because the key pair is mathematically related, whatever is encrypted with a Public Key may only be decrypted by its corresponding Private Key and vice versa. As the public key is fed the cloud exchange, the private key must be fed to the device. Note that the cloud exchange doesn’t accept private keys.
  • Public Key Format – Cloud Exchange supports four formats of public key, RSA_PEM, RSA_X509_PEM, ES256_PEM and ES256_X509_PEM.

When creating a device, the device type plays a very important role of directing to the right dashboards, each dashboard is custom made and fine tuned to suit the application. Our dashboards range from energy monitoring to monitoring comforts conditions.
