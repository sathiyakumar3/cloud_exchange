The device is identified by the following parameters.
Device ID - It is the unique ID that you are going to identify your device.
Registry ID - It is basically the site name, under which you are expecting the device to be placed.
Along with the above credentials, the following parameters are also required security.
Public and Private Key Public keys are generally opened to the public, as the name suggests. On the other hand, the Private Key must remain confidential to its respective owner. Because the key pair is mathematically related, whatever is encrypted with a Public Key may only be decrypted by its corresponding Private Key and vice versa. As the public key is fed the cloud exchange, the private key must be fed to the device. Note that the cloud exchange doesn’t accept private keys.
Public Key Format Cloud Exchange supports four formats of public key, RSA_PEM, RSA_X509_PEM, ES256_PEM and ES256_X509_PEM.
